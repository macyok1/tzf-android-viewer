# Редизайн проектного workspace и сшивки

## Цель

Убрать параллельные режимы `primary/secondary` и сделать дерево проекта единственным источником истины для загрузки, видимости, трансформаций, группировки и регистрации сканов.

## Компоновка

Используется утверждённый вариант B:

- viewport занимает всё доступное пространство;
- слева расположена вертикальная CAD-панель `＋`, `⌂`, `R/M`, `◎`, `⋯`;
- справа снизу показывается компактная карточка выбранной пары и команда точной сшивки;
- дерево и параметры открываются как взаимоисключающие overlay-панели;
- overlay занимает `min(220dp, 35% viewport)`;
- обе Activity фиксируются в `portrait`.

Старые кнопки «Опорный скан», «Перемещаемый скан» и отдельное добавление из меню сшивки удаляются.

## Пакетное добавление

Кнопка `＋` запускает системный picker с `ACTION_OPEN_DOCUMENT`, `CATEGORY_OPENABLE`, типом `*/*`, фильтрацией расширения `.tzf` и `EXTRA_ALLOW_MULTIPLE=true`.

Каждый уникальный URI:

1. получает persistable read permission;
2. создаёт отдельный `ProjectModel.Scan`;
3. получает собственные generation, preview session и GPU-чанки;
4. загружается общей последовательной очередью;
5. отображает состояние `ожидает / загружается / готов / ошибка` в дереве.

Повторно выбранный URI не создаёт дубликат, а переводит существующий недоступный скан на новый URI.

## Дерево и связки

Обычное касание строки выбирает узел для просмотра свойств. Долгое нажатие включает множественный выбор. В этом режиме доступны:

- «Связать» — создать группу из выбранных узлов;
- «Переместить» — reparent в выбранную группу;
- «Отмена» — выйти без изменения дерева.

Группировка и reparent сохраняют мировое положение узлов. Меню `⋮` строки содержит переименование, отсоединение, расформирование группы и удаление с подтверждением. Удаление узла из проекта не удаляет исходный TZF.

Строка содержит отдельные элементы:

- видимость;
- имя и состояние загрузки;
- роль `R`;
- роль `M`;
- меню `⋮`.

## Роли регистрации

Роль назначается любому `Scan` или `Group`. В проекте одновременно существуют не более одного reference `R` и одного moving `M`. Повторное нажатие активной роли снимает её; назначение роли другому узлу переносит её.

Запрещены:

- один узел одновременно в ролях `R` и `M`;
- регистрация узла с самим собой;
- регистрация группы с её предком или потомком.

Идентификаторы выбранных узлов сохраняются в проекте. После удаления выбранного узла соответствующая роль очищается.

## Gizmo и трансформации

Gizmo отображается на мировом pivot узла `M` и изменяет только его локальные X/Y/Z/RZ. Для группы движение применяется к трансформации группы, поэтому все потомки перемещаются как единое целое. Камера никогда не записывает изменения в узел проекта.

Renderer получает мировые трансформации исключительно из `ProjectModel.Node.worldTransform()`. Старый фиксированный `secondaryTransform` удаляется после миграции регистрации и измерения.

## Точная сшивка

Команда активна только при корректной паре `R/M` и готовых данных обоих узлов.

- Для скана используется его равномерная preview-выборка.
- Для группы выборка собирается из всех видимых готовых потомков после применения их мировых трансформаций.
- Вклад потомков ограничивается квотой, чтобы один большой скан не вытеснял остальные.
- ICP оптимизирует только мировые X/Y/Z/RZ узла `M`.
- Принятый мировой результат переводится в локальную трансформацию относительно родителя `M`.
- Ненадёжный результат не применяется.

Карточка пары показывает тип узла, имя и число сканов. Во время расчёта команда заменяется прогрессом и кнопкой отмены.

## Команда «Показать всё»

`⌂` вычисляет мировые AABB всех видимых готовых облаков, включая трансформации групп, и устанавливает центр камеры и zoom так, чтобы объединённая граница помещалась во viewport с 10% полем. Команда не меняет трансформации узлов.

При отсутствии видимых готовых облаков команда ничего не меняет и показывает краткий статус.

## Сохранение

Формат проекта повышается до версии 2 и сохраняет:

- `referenceNodeId` и `movingNodeId`;
- раскрытые группы;
- URI, видимость, цвет, трансформации и иерархию;
- состояние камеры, сетки, бюджета и толщины точек.

Decoder сохраняет совместимость с форматом 1, назначая новые поля значениями по умолчанию.

## Порядок реализации

1. Модель выбора ролей, формат проекта v2 и тесты.
2. Пакетный picker и очередь загрузки.
3. Новое дерево, множественный выбор и операции групп.
4. Вертикальная компоновка B и компактная карточка пары.
5. Gizmo для выбранного узла `M`.
6. Регистрация scan/group через выбранные `R/M`.
7. Настоящий fit-to-visible для `⌂`.
8. Удаление legacy `primary/secondary` UI и renderer path.
9. Portrait lock, полная сборка и проверка устройства.

## Проверка

- загрузить одним выбором минимум три TZF;
- создать, переименовать, переместить и расформировать связку без скачка облаков;
- назначить и сохранить роли для scan↔scan, group↔scan и group↔group;
- переместить `M` gizmo и проверить неизменность `R`;
- выполнить и отменить ICP;
- проверить `⌂` для одного, нескольких и скрытых облаков;
- закрыть приложение принудительно и восстановить проект;
- повернуть устройство и убедиться, что Activity остаётся портретной;
- пройти JVM tests, Android debug build, portable C++ tests и GitHub Actions.
