# Встроенное обновление из GitHub

## Цель

Пользователь обновляет TZF Viewer без USB и ADB: открывает `⋯`, нажимает «Проверить обновление», приложение загружает свежий APK из публичного GitHub Release и передаёт его системному установщику Android.

## Подпись и версии

Все обновляемые APK подписываются одним release-keystore. Keystore и пароли хранятся только в GitHub Actions Secrets; в репозиторий попадают лишь Gradle-конфигурация и имена переменных окружения.

CI задаёт монотонный `versionCode` из `GITHUB_RUN_NUMBER` и `versionName` вида `0.2.<run-number>`. Локальная сборка без release-секретов остаётся debug-сборкой и не публикуется как обновление.

Переход с уже установленного ephemeral debug APK на постоянную release-подпись требует однократного удаления старого приложения и ручной установки первого release APK. Все последующие версии устанавливаются поверх него.

## Публикация

После тестов workflow собирает `assembleRelease`, вычисляет SHA-256 APK и публикует в prerelease `nightly` два постоянных asset:

- `tzf-viewer-latest.apk`;
- `update.json`.

`update.json` содержит `versionCode`, `versionName`, прямой HTTPS URL APK, SHA-256 и минимальную версию Android. Assets заменяются только после успешных JVM, Android и portable C++ tests.

## Клиент обновления

Изолированный `AppUpdater`:

1. по явному нажатию загружает `update.json` с таймаутами;
2. сравнивает опубликованный `versionCode` с установленным;
3. показывает текущую и доступную версии;
4. после подтверждения скачивает APK в приватный cache-каталог;
5. вычисляет SHA-256 и требует точного совпадения с manifest;
6. передаёт APK системному установщику через `FileProvider` и временное read-разрешение.

Если приложению не разрешена установка неизвестных приложений, открывается системный экран `ACTION_MANAGE_UNKNOWN_APP_SOURCES`. Пользователь возвращается и повторно нажимает установку. Тихая установка не используется.

## UI

В панели `⋯` добавляется одна строка:

- «Проверить обновление» в обычном состоянии;
- прогресс загрузки во время скачивания;
- «Установить <version>» после успешной проверки.

Ошибки сети, JSON, хеша, разрешения и системного установщика выводятся в существующий status без закрытия проекта.

## Безопасность

- только HTTPS URL на `github.com` или `objects.githubusercontent.com`;
- обязательная проверка SHA-256 до запуска installer;
- `FileProvider` экспортирован=false и предоставляет только cache-подкаталог обновлений;
- APK невозможно применить поверх приложения с другой подписью;
- redirect на неразрешённый host отклоняется;
- загруженный временный APK заменяется атомарно и удаляется при следующей проверке.

## Проверка

- unit tests парсинга manifest, сравнения версий, host allowlist и SHA-256;
- debug APK, release APK и Android lint;
- GitHub Actions публикует оба asset и корректный hash;
- первая release-установка выполняется вручную после удаления старой debug-сборки;
- следующая CI-версия обнаруживается кнопкой и устанавливается поверх предыдущей с сохранением проектов;
- отказ сети, неверный hash и запрет unknown sources дают понятное сообщение и не запускают installer.
