# Настройки приложения и навигация к проектам

## Цель

Глобальные функции приложения должны находиться вне рабочего меню сшивки. Пользователь должен явно возвращаться из workspace к списку проектов, а обновление должно находиться на отдельном экране настроек и всегда иметь корректное кликабельное состояние.

## Экран проектов

В заголовке `ProjectsActivity` размещаются:

- название «Проекты»;
- кнопка `⚙` с content description «Настройки приложения»;
- кнопка «＋ Новый проект».

`⚙` открывает `SettingsActivity`. Возврат из настроек сохраняет список проектов без изменений.

## Экран настроек

`SettingsActivity` содержит:

- кнопку `←` и заголовок «Настройки»;
- текущие `versionName` и `versionCode`;
- канал обновлений `GitHub nightly`;
- основную кнопку состояния updater;
- текстовый статус операции.

Состояния кнопки:

1. `Проверить обновление` — включена;
2. `Проверяем…` — временно выключена;
3. `Скачать <version>` — включена;
4. `<percent>%` — временно выключена;
5. `Установить <version>` — включена;
6. `Повторить` после любой ошибки — включена.

Если доступной версии нет, кнопка возвращается в состояние `Проверить обновление`. Если требуется разрешение unknown sources, после возврата из системных настроек APK остаётся доступным по кнопке `Установить`.

Updater и его UI-state удаляются из `MainActivity`. `AppUpdater` остаётся общим изолированным компонентом и используется только `SettingsActivity`.

## Возврат из workspace

В верхней панели `MainActivity` слева от названия проекта добавляется кнопка `←` с content description «К проектам».

Она:

1. завершает активные UI-взаимодействия;
2. сохраняет проект через существующий `ProjectStore`;
3. завершает `MainActivity` через `finish()`;
4. возвращает уже существующую `ProjectsActivity` из back stack без создания дубликата.

Системная кнопка Android «Назад» выполняет тот же путь. Если открыта overlay-панель, первое нажатие закрывает панель, второе возвращает к проектам.

## Проверка

- `⚙` открывает отдельные настройки с правильной версией;
- updater-кнопка кликабельна сразу после открытия;
- сетевой отказ возвращает активную кнопку `Повторить`;
- успешная проверка проходит состояния скачать/прогресс/установить;
- возврат из unknown-sources settings сохраняет готовый APK;
- updater отсутствует в параметрах сшивки;
- `←` сохраняет проект и возвращает к существующему списку;
- системный Back сначала закрывает overlay, затем возвращает к проектам;
- JVM tests, Android lint, debug/release APK и GitHub Actions проходят.
