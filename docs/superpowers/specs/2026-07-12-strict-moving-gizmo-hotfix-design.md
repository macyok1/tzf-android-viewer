# Строгая привязка gizmo только к M

## Цель

Вернуть простое поведение первых версий: манипулятор управляет исключительно выбранным узлом `M` по `X/Y/Z/RZ`.

## Инварианты

- При отсутствии `M` gizmo не отображается и не принимает touch.
- Единственный источник выбранной цели — `project.movingNodeId`.
- Renderer получает один явный target: id узла `M`, ids его отображаемых сканов и мировую pose `M`.
- Drag возвращает новую мировую pose только для этого target.
- `MainActivity` принимает результат только если id target совпадает с текущим `project.movingNodeId`.
- Для одиночного `M` меняется только его локальная pose.
- Для группы `M` меняется только pose группы; pose потомков не переписываются.
- Pose `R`, остальных узлов, camera state, scene frame и сетки не меняются.
- Допустимы только `XYZ` и `RZ`; `RX/RY` отсутствуют.

## Упрощение renderer

Набор `activeSceneIds` больше не является идентичностью редактируемого объекта. Он используется только для определения отображаемых потомков группы. Идентичность цели хранится отдельным `targetNodeId` и передаётся обратно вместе с результатом drag.

Асинхронный callback от старой цели игнорируется после смены `M`. Это исключает применение задержавшегося GL-события к новому узлу или к `R`.

Pivot gizmo равен мировой позиции target `M`. Движение pivot во время drag следует pose `M`, не вызывая пересчёт общего scene frame.

## Проверка

- Выбрать голубой скан как `R`, оранжевый как `M`; все ручки меняют только оранжевый.
- После начала drag сменить роли; задержавшийся callback не меняет новый `M` или `R`.
- Camera orbit вне ручек не меняет ни один узел.
- Группа `M` движется целиком, дочерние локальные pose неизменны.
- Снять роль `M`: gizmo исчезает.
- JVM tests, debug APK для четырёх ABI и lint проходят.
