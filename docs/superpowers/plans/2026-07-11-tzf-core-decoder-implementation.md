# План реализации независимого декодера TZF

## Этап 1. Воспроизводимый анализ структуры

1. Зафиксировать тестовый файл и его размер.
2. Прочитать основной заголовок и четыре известные tail-ссылки.
3. Проверить поля tail-секции каталога: значение 1764, число дескрипторов 36 и начало массива.
4. Разобрать 36 дескрипторов `u64 offset + u32 size` и проверить гипотезу о 12 тройках.
5. Сопоставить дескрипторы с ETW-чтениями и классифицировать пустые, повторяющиеся и выходящие за границы записи.

## Этап 2. Переносимое ядро и CLI

1. Добавить платформонезависимый модуль `tzf-core` на C++20.
2. Реализовать безопасное little-endian чтение и проверки переполнения/границ.
3. Реализовать модели заголовка, tail-ссылок, записи столбца и трёх блоков.
4. Добавить CLI-команды для сводки файла, вывода записи и извлечения блока.
5. Собирать тот же исходный код на Windows и через Android NDK.

## Этап 3. Автоматические проверки

1. Проверить известные значения `Scan_001.tzf`.
2. Проверить, что все непустые дескрипторы находятся внутри файла.
3. Добавить синтетические тесты для обрезанных данных, переполнений и неверных размеров записей.
4. Сохранить машинно-читаемую сводку для сравнения между итерациями.

## Этап 4. Восстановление цепочки кодеков

1. Снять сигнатуры и статистику каждого из трёх типов блоков.
2. Найти метаданные компонентов и сопоставление GUID с реализациями RealWorks.
3. Реализовать по одному этапу декомпрессии/transpose/derive с сохранением промежуточных результатов.
4. Определить маску валидности и число точек выбранного столбца.
5. Восстановить XYZ и сравнить с oracle RealWorks.

## Этап 5. Расширение и Android

1. Обобщить декодирование на несколько столбцов и тестовых файлов.
2. Добавить цвет, интенсивность и метаданные.
3. Подключить `tzf-core` к JNI потоковым API партий точек.
4. После подтверждения корректности перейти к визуализации облака.

## Условия первого завершённого результата

- CLI независимо читает карту файла и таблицу блоков.
- Один столбец полностью декодируется в XYZ.
- Количество точек и координаты совпадают с RealWorks в пределах погрешности `float32`.
- Все установленные структуры и преобразования описаны в `TZF_FORMAT.md` с доказательствами и уровнем уверенности.
